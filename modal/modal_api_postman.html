Accesos API

Haz clic aquí para ingresar a la creación del token

Damos clic en el icono del ojo para que el token sea desbloqueado y poder copiarlo.

Copiamos el texto y lo llevaremos a la pagina de apidocs. Ha esta pagina llegamos ingresando la palabra apidocs en la url del cliente ejemplo: demo.buk.cl/apidocs.

Debemos hacer clic en Autorizar para que ingresemos nuestro token copiado anteriormente.

  • Debemos ver que el país este correcto

Pegamos nuestro token creado.

Hacemos clic en Authorize

Empleados

  • Vamos a seleccionar la opción y aquí encontraremos cada uno de los endpoint que se encuentran disponibles en la categoria empleados.

/employees/{id}

  • En este caso probaremos con este endpoint que es para traer los datos de un colaborador en especifico


GET Los metodos GET sirven para obtener datos del servidor ejemplo, consultar todos los empleados, consultas algún empleado especifico, consultas vacaciones, etc
POST Los metodos POST sirven para enviar datos al servidor y crear un recurso ejemplo, crear empleados, licencia, inasitencia, items,etc
PATCH Los metodos PATCH sirven para aplicar modificaciones parciales a un recurso existente ejemplo, modificar la dirección a un colaborador existente, modificar los días a una licencia anteriormente creada en la plataforma,etc
DELETE Los metodos DELETE sirven para solicitar al servidor eliminar un recurso ejemplo, eliminar un colaborador, eliminar una licencia, etc
  • Ahora debemos ingresar el parametro que nos solicita el metodo, puede ser el rut sin puntos ni guión o el id del colaborador que buscaremos.
  • Por último, presionamos el bóton Pruébalo!

1 URL: Corresponde al endpoint que vamos a utilizar para nuestra consulta y que tipo es, en este caso es un metodo GET
2 Clase de la respuesta: Nos indica el código de respuesta que debemos recibir si la consulta se logra con exito!. Si no recibimos el código 200, es por que tenemos un inconveniente y pueden ser:
- Dato no encontrado
- Se acabo el tiempo de espera
- Error de código interno
3 Valor Ejemplo/ modeloAquí veremos un ejemplo del JSON (formato de dato) que nos retornara el sistema. Si damos click en donde dice modelo, podremos ver cada uno de los atributos (datos que componen el json) y veremos de que tipo es, si es un entero (integer), texto (string), verdadero o falso (boolean), objeto (empleado, licencia, etc).
4 Párametros Veremos los datos que podemos ingresar en nuestro endpoint, en algunos casos tendremos más o menos, dependera del endpoint a utilizar. No todos los parametros son obligatorio, dependera de la documentación.
  • Luego de enviar nuestra petición haciendo clic en el bóton Pruébalo! recibiremos la respuesta en formato json.
  • Ahora es momento de realizar este proceso, desde la herramienta Postman.

1 Curl: Es una herramienta de línea de comandos que te ayuda a descargar o enviar datos desde y hacia servidores
2 La URL de solicitud: Es la dirección exacta del endpoint dentro del servidor que nos va a permitir realizar acciones específicas, como obtener información, crear, actualizar o eliminar dato.
3 Cuerpo de la respuesta: Aquí tendremos los datos recibidos desde el servidor en formato JSON
4 Código de la respuesta: Nos indica mediante un código el exito! de nuestra consulta o el error en su defecto.

New

Una vez abierta la herramienta Postman, daremos clic en New para crear una conexión.

HTTP

Seleccionamos HTTP que es el protocolo que nos sirve para utilizar la API.

  • Volvemos a la apidocs y copiamos URL de la solicitud para poder pegarla en Postman

1

Aquí debemos pegar la URL copiada anteriormente en la pagina de apidocs

2

Veremos que en este punto (Query Params) habilita una key llamada rut con un value. Este valor corresponde en este caso al rut del colaborador, esto pasa por que la url que nosotros copiamos en el paso 1, tiene como parametro el rut que consultamos en la apidocs y Postman lo respresenta de esta manera

Headers

Ahora debemos hacer clic en Headers para poder entregarle nuestro token y así tener los permisos necesarios al momento de realizar la consulta desde Postman.

1

Aquí vamos a crear nuestro parametro a diferencia de la imagen anterior, donde con solo pegar la url se nos creo el parametro ahora debemos crearlo nosotros. En Key escribimos auth_token y en value pegamos el token que copiamos desde BUK al comienzo del manual.

Send
